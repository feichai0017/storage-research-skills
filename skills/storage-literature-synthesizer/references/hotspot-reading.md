# Hotspot Reading

Use this when the user asks:
- what is hot in storage / databases / distributed systems?
- what directions are emerging?
- where should we look for ideas?

## Reading Rule

Do not answer from memory alone.
Read recent accepted-paper lists, conference programs, CFPs, and workshop lists.

## Clustering Rule

Cluster by recurring pressure, not by keyword.

Bad clusters:
- "AI"
- "agents"
- "vector"
- "distributed systems"

Good cluster properties:
- anchored in repeated primary-source signals
- reducible to one concrete systems pressure
- narrow enough to map to a plausible artifact class
- distinguishable from mere vocabulary drift

## Reading Scope Rule

This file does not provide preselected hotspot examples.
Use it to build a scoped reading plan from fresh signals, not to reuse canned topic lists.

For each candidate cluster:
- record the source venue or program
- record the repeated systems pressure
- record the likely artifact class
- record the closest adjacent systems or papers to compare

## Output Shape

For each hotspot, return:
- signal source
- repeated papers or venue signals
- underlying systems pressure
- why it matters to storage/database researchers
- 2-3 candidate idea directions
