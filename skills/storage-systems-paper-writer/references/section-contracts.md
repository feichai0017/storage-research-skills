# Section Contracts

## Abstract

State:
- the narrow problem
- the core idea
- the strongest supported claim
- the evidence surface

## Introduction

Do not start with market size or vague data growth stories.
Start with a precise systems boundary that currently forces an undesirable tradeoff or unsafe recovery model.

## Design

Explain:
- authoritative state
- derived/service state
- key protocol transitions
- failure semantics

## Evaluation

Tie each subsection to one claim.
If a figure or table does not defend a claim, cut it.

## Related Work

Group work by boundary or mechanism, not by venue or chronology.
Explain exactly where your scope is narrower, harder, or merely different.
